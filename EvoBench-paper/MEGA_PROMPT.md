### 📝 3. 顶会级论文构写标准 (针对阶段组 H)
* **Sushi, not Curry (聚焦原则)：** 论文只能有 1-2 个基于你激进Idea的核心创新点，其余部分极致严谨。严禁堆砌无用模块。
* **Figure 1 霸权：** 初稿前构思好"图 1"。图 1 必须能独立传达这篇论文的最核心贡献，在 tex 中留下极为详尽的视觉元素 prompt。
* **强制消融实验 (Ablations)：** 论文提到的有效组件，必须有移除该组件后的对比数据，否则退回实验阶段。
* **强基线 (Strong Baselines)：** 基线模型必须受到同等超参数调优对待。
* **字数防卫：** 严守长度底线，字数不足必须扩写实质细节，**严禁车轱辘话凑字数**。

### 🧐 4. 证据与相关性红线审查 (针对阶段 27 & 32)
* **一致性核查：** Draft 与 `results.json`/Log 逐行比对。若宣称跑了10种数据集实际只有2种，直接判定为 **CRITICAL FABRICATION (重大伪造)**，强制退回实验阶段。
* **文献保真：** 提取卡片必须保留原版 `cite_key` 和 DOI。拒绝对领域无关的论文。同时，每一篇文献必须使用tavily检索到文献本体、确认所有信息的真实性并打钩，严禁编造或篡改文献信息。
* **相关性审查：** 论文中每一处引用都必须与内容高度相关，且必须在文献综述中清晰地说明与现有工作的区别和联系，严禁为了增加参考文献数量而引用无关或边缘相关的工作。
* **第三方审查：** 在第33阶段，必须调用单独上下文大模型进行最严苛的外部专家评审，你可以启动子任务或者使用命令行调用claude code来实现。审查必须覆盖论文的创新性、实验的真实性和充分性、以及文献引用的相关性和准确性。在得到审查意见后，必须返回对应阶段进行针对性的优化，重走相关阶段的流程。

### 🛠️ 5. 环境与库兼容性规范 (针对阶段 15)
* **沙盒依赖：** 优先 Python stdlib, numpy, math, statistics。
* **NumPy 2.x 强制兼容 (CRITICAL)：**
    * 废弃 `np.trapz` → 强制使用 `np.trapezoid`
    * 废弃 `np.erfinv` → 强制使用 `scipy.special.erfinv`
    * 废弃 `np.bool`, `np.int`, `np.float` → 强制使用 Python 原生类型 `bool, int, float`
    * 废弃 `np.math` → 强制使用标准库 `math`
